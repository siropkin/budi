## What was done in session 27 (2026-03-13)

### 1. Graph self-reference bug fix — DONE (f56e85b)
- Root cause: `extract_reference_tokens()` processed definition lines that matched
  `looks_like_reference_line()` (e.g., `def dispatch_request(self, ctx) -> ft.ResponseReturnValue:`
  has '.' and '('), extracting the definition name as a reference token
- This caused definition chunks to self-reference in `graph_token_to_chunk_ids`,
  then `diversify_channel_by_path` (1-per-file dedup) dropped the actual caller
  when both were in the same file
- Fix: skip `looks_like_definition_line()` lines in `extract_reference_tokens()`
- 5 new tests added

### 2. Graph-aware sym-use no-call-site skip — DONE (5eb56e1)
- `sym_use_no_call_site_skip` only checked top 3 candidates for call expressions
- Real callers often ranked below top 3 due to HNSW variance
- Fix: also check all candidates with graph > 0 (callers by construction)
- Flask P8: ctx went from 0 → 723 chars, benchmark went from 89% → 100%
